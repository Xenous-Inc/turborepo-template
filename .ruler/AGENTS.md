# AGENTS.md

This file provides guidance to AI coding agents when working with code in this repository.

## Project Overview

Turborepo monorepo ("xenous") with three apps sharing packages and tooling. Uses pnpm workspaces with version catalogs (`pnpm-workspace.yaml`). Node.js >=24.11, pnpm ^10.

## Architecture

The server app (`apps/server`) is the API layer ‚Äî both frontends talk to it. It uses Nitro + H3 for HTTP and exposes type-safe oRPC procedures that the frontends consume via `@orpc/client` + TanStack Query. Auth is handled by Better Auth with Drizzle adapter; auth tables are generated into `packages/db/src/schema/auth.ts`.

- **`apps/server`** ‚Äî Nitro API. oRPC routers in `src/routers/`, Nitro routes in `src/routes/`. `publicProcedure` and `protectedProcedure` in `src/utils/orpc.ts`; context (`session`, `user`, `db`) in `src/utils/context.ts`.
- **`apps/web`** ‚Äî Next.js 15 (App Router, Turbopack).
- **`apps/tanstack`** ‚Äî TanStack Start (TanStack Router, Vite).
- **`packages/db`** ‚Äî Drizzle ORM schema + client + migrations. Auth tables auto-generated by Better Auth.
- **`packages/ui`** ‚Äî shadcn/ui + Tailwind CSS 4. Add components via `pnpm ui-add`.
- **`packages/validators`** ‚Äî Shared Zod 4 schemas (`model/` and `payload/`).
- **`packages/logger`** ‚Äî Consola singleton.

## Validation

- Assume dev servers are already running. Never build apps to validate code.
- To validate TypeScript or formatting, use predefined scripts: `pnpm run <script> -F <package_or_app>` (e.g. `pnpm run typecheck -F server`, `pnpm run check:fix -F @xenous/ui`).
- Check `package.json` at root and per-package for all available scripts before running any commands.

## Key Conventions

- Biome for linting+formatting (not ESLint/Prettier). Config in `tooling/biome/`. Run `pnpm check:fix` to auto-fix.
- `import type` required for type-only imports (`verbatimModuleSyntax`).
- No enums ‚Äî use `as const` objects. No non-null assertions.
- Env vars validated at runtime via `@t3-oss/env-*`. Each app has `.env.local.example`.
- Conventional commits enforced by commitlint via Lefthook.
- Shared dependency versions managed through pnpm catalogs. Use `"catalog:"` or `"catalog:<name>"` in package.json.
- Scaffold new packages with `pnpm turbo gen init`.
- Use `pnpx` instead of `npx` to execute package binaries.
- In React components, separate logical groups (hooks, derived state, handlers, render) with blank lines for readability.
- Read existing code in the area you're modifying before making changes ‚Äî match surrounding patterns and style.
